# ADR-004: Дизайн Gamification Engine

**Статус:** Принято

**Дата:** 2026-07-12

## Контекст

Приложению нужна система геймификации для мотивации пользователей. Система должна обрабатывать XP, уровни, бейджи, streak'и и leaderboard — всё в едином тестируемом компоненте.

## Решение

Принять единый класс **GamificationEngine**, управляющий всей логикой геймификации:

### Формула XP
```
XP = base_xp (100) + score_bonus (50 если score >= 90) + streak_bonus (200 если streak >= 3)
```

### Формула уровня
```
level = total_xp // 1000 + 1
```

### Бейджи (всего 8)
| Бейдж | Критерий |
|-------|----------|
| first_session | Завершить 1 сессию |
| streak_3 | 3-дневный streak |
| streak_7 | 7-дневный streak |
| empathy_master | empathy >= 90 |
| problem_solver | problem_solving >= 90 |
| quick_learner | Завершить 3 сессии за один день |
| top_performer | overall_score >= 95 |
| dedicated | Всего 30 сессий |

### Leaderboard
- Топ-10 пользователей по total XP (descending)
- Обновляется при каждой XP транзакции

## Рассмотренные альтернативы

- **Внешний сервис геймификации:** Добавил бы сетевую зависимость — отклонено из-за принципа in-memory-first.
- **Геймификация через БД (SQL триггеры):** Сложно тестировать и версионировать — отклонено.
- **Отдельные сервисы на каждую фичу (XP, badges, streaks):** Over-engineering для текущего масштаба — отклонено ради простоты.

## Последствия

- **Положительно:** Единый тестируемый класс
- **Положительно:** Все правила геймификации явные и документированные
- **Положительно:** Нет внешних зависимостей
- **Отрицательно:** Добавление новых типов бейджей требует изменения кода
- **Отрицательно:** Leaderboard — O(n) по количеству пользователей — приемлемо для MVP (<10k пользователей)
