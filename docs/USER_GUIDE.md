# Руководство пользователя — AI Roleplay Coach Hub

> Для операторов, тренеров и администраторов. Пошаговые инструкции для всех ролей.

---

## 1. Введение

**AI Roleplay Coach Hub** — симулятор для отработки навыков общения с «трудными» клиентами.
Система создаёт AI-клиента с заданным психотипом, проблемой и триггерами, оценивает
диалог по 6 измерениям (эмпатия, структура, решение проблем и др.), начисляет XP
и бейджи, а также выявляет статистические перекосы (fairness).

### 1.1 Для кого этот документ

| Роль | Кто это | Что делает в системе |
|------|---------|---------------------|
| **Оператор** | Сотрудник кол-центра | Проходит симуляции, получает оценку, копит XP и бейджи |
| **Тренер** | Методист / супервайзер | Создаёт сценарии, назначает учебные планы, смотрит статистику группы |
| **Администратор** | DevOps / Team Lead | Управляет пользователями, проверяет fairness-отчёт, видит дашборд |

### 1.2 Основные понятия

| Термин | Определение |
|--------|-------------|
| **Сессия** | Диалог с AI-клиентом по заданному сценарию |
| **Сценарий** | Описание роли AI-клиента: психотип, проблема, триггеры, сложность |
| **Оценка (evaluation)** | Автоматическая оценка диалога Coach-агентом по 6 измерениям |
| **Sandwich feedback** | Формат обратной связи: позитив → зоны роста → позитив |
| **XP** | Опыт за завершённые сессии и достижения |
| **Уровень (level)** | `level = XP_total / 1000 + 1` |
| **Бейдж** | Значок за особое достижение (всего 8) |
| **Streak** | Количество дней подряд с активностью |
| **Leaderboard** | Таблица рейтинга по XP |
| **Учебный план** | Последовательность сценариев, назначенная тренером |
| **DDA** | Dynamic Difficulty Adaptation — сложность AI-клиента растёт после успешных сессий |
| **Fairness** | Метрики справедливости: нет ли перекоса по полу/возрасту/акценту |
| **Куратор (Curator)** | AI-агент, генерирующий сценарии и вопросы для quiz |

### 1.3 Как читать этот документ

- **Оператору** → разделы 2 (быстрый старт), 3 (работа оператора), 6 (сценарии)
- **Тренеру** → разделы 2, 4 (работа тренера), 6
- **Администратору** → разделы 2, 5 (работа администратора), 6
- **Всем** → раздел 7 (FAQ) и 8 (troubleshooting)

## 2. Быстрый старт

### 2.1 Регистрация

**Через API (для тестирования):**
```bash
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username": "operator1", "email": "op1@example.com", "password": "StrongPass1"}'
```

**Через фронтенд:**
1. Открой `http://localhost:5173`
2. Нажми **Register** (или перейди на `/register`)
3. Заполни поля:

| Поле | Описание | Пример |
|------|----------|--------|
| Username | Логин (уникальный) | `operator1` |
| Email | Email (уникальный) | `op1@example.com` |
| Password | Пароль (мин. 8 символов, заглавная + цифра) | `StrongPass1` |

4. Нажми **Submit** → получишь access token (30 мин) + refresh token (7 дней)
5. Автоматический редирект на `/sessions`

### 2.2 Вход

Форма `/login`:
```bash
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "operator1", "password": "StrongPass1"}'
```

При успехе: access token в ответе, refresh token в cookies (httpOnly).

### 2.3 Первые шаги после входа

1. Посмотри список доступных сценариев на странице `/sessions/new`
2. Выбери **Confused client** (рекомендуется для новичков)
3. Начни сессию — AI-клиент поприветствует тебя
4. Отправь 3-5 сообщений, затем нажми **Finish**
5. Посмотри оценку на странице `/sessions/:id/evaluate`
6. Зайди в профиль `/profile` — проверь XP и бейджи

## 3. Руководство оператора

### 3.1 Запуск сессии

**Пошаговая инструкция:**
1. На главной панели нажми **New Session** (зелёная кнопка, `/sessions/new`)
2. Выбери сценарий из списка:

| Сценарий | Сложность | Описание |
|----------|-----------|----------|
| Confused client | Лёгкая | Клиент не понимает продукт, нужна помощь |
| Aggressive client | Средняя | Клиент зол, нужно валидировать эмоции |
| Demanding client | Сложная | Клиент требует невозможного, нужен structure response |
| Твои предыдущие сценарии | Разная | Повторение слабых мест |

3. Ознакомься с краткой информацией о сценарии (психотип, проблема, триггеры)
4. Нажми **Start** — через секунду AI-клиент отправит первое сообщение
5. Интерфейс чата: поле ввода внизу, история сообщений посередине, таймер слева

### 3.2 Ведение диалога

**Механика:**
- Каждое сообщение отправляется через `POST /api/v1/sessions/{id}/turns`
- AI-клиент отвечает синхронно (обычно 2-5 секунд)
- Чат поддерживает Markdown (жирный, списки) — используй для структурирования ответов
- Длина сообщения: до 2000 символов
- Сессия не может длиться более 60 минут

**DDA (Dynamic Difficulty Adaptation):**
- После 2+ успешных сессий (score >= 70) клиент становится сложнее
- После 3+ неудачных сессий сложность снижается
- Это гарантирует, что ты всегда работаешь на границе своих возможностей

**Советы по каждой теме:**

| Тема клиента | Что работает | Что не работает |
|-------------|--------------|-----------------|
| Жалоба (aggressive) | Эмпатия, валидация эмоций, структура решения | Спор, игнорирование эмоций |
| Непонимание (confused) | Простые слова, аналогии, шаги | Жаргон, раздражение |
| Завышенные требования (demanding) | Чёткие границы, альтернативы, эскалация | Обещания, уход от ответа |

### 3.3 Завершение и оценка

**Два способа завершить:**
1. Нажми **Finish** в интерфейсе — сессия завершится, Coach-агент начнёт оценку
2. Если сессия прервалась (таймаут, закрытие вкладки) — оценка не будет создана

**Процесс оценки:**
1. `POST /api/v1/sessions/{id}/evaluate` — запустить оценку вручную (если не запустилась авто)
2. Coach-агент анализирует весь диалог по 6 измерениям:

| Измерение | Что оценивает | Вес |
|-----------|--------------|-----|
| empathy | Эмпатия, валидация эмоций | 20% |
| listening | Активное слушание, уточнение | 15% |
| problem_solving | Решение проблемы, альтернативы | 25% |
| handling_aggression | Работа с агрессией | 20% |
| communication | Чёткость, вежливость | 10% |
| structure | Структура ответа | 10% |

3. Каждое измерение: 0-100 баллов → итоговый weighted score
4. Sandwich feedback: сильные стороны → зоны роста → рекомендации

**Шкала:**
| Итоговый score | Уровень |
|----------------|---------|
| 90-100 | Отлично |
| 70-89 | Хорошо |
| 50-69 | Удовлетворительно |
| <50 | Нужно улучшать |

### 3.4 Прогресс: XP, уровни, бейджи

**XP начисляются за:**

| Действие | XP |
|----------|----|
| Завершение сессии с оценкой | 100 XP |
| Score >= 90 | +50 XP |
| Streak 3+ дней | +200 XP |
| Получение бейджа | +50 XP |

**Уровни:**
```
level = total_xp / 1000 + 1
```
Пример: 4500 XP → уровень 5 (4500/1000 + 1 = 5.5 → floor = 5)

**8 бейджей:**

| Бейдж | Как получить |
|-------|-------------|
| 🏆 First Session | Завершить первую сессию |
| 🏆 Perfect Score | Получить score >= 95 |
| 🏆 Streak Master | Достичь streak 5 дней |
| 🏆 Level 5 | Достичь уровня 5 |
| 🏆 Level 10 | Достичь уровня 10 |
| 🏆 Level 20 | Достичь уровня 20 |
| 🏆 Aggressive Handler | Завершить 5 агрессивных сценариев |
| 🏆 Empathy Expert | Средняя эмпатия > 80 за 10 сессий |

**Где смотреть:**
- `/profile` — твой профиль: XP, уровень, бейджи, streak
- `/leaderboard` — таблица рейтинга (топ-100 по XP)

### 3.5 Учебные планы

Если тренер назначил тебе учебный план:
1. Зайди на `/sessions/plan` — увидишь список назначенных сценариев
2. Проходи их по порядку (рекомендуется)
3. После каждого — смотри оценку и рекомендации
4. После завершения плана — тренер увидит статистику

### 3.6 Типовые проблемы оператора

| Проблема | Решение |
|----------|---------|
| Не знаю, что ответить | Используй структуру: эмпатия → понимание → решение |
| Клиент агрессивен | Валидируй: «Я понимаю ваше недовольство...» |
| Низкий score по empathy | Добавляй фразы понимания перед решением |
| Низкий score по structure | Используй нумерацию: «Во-первых..., во-вторых...» |

## 4. Руководство тренера

> Требуется роль: TRAINER или ADMIN.

### 4.1 Управление сценариями

**Создание сценария вручную:**
```bash
curl -X POST http://localhost:8000/api/v1/scenarios \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "title": "Возврат товара",
    "description": "Клиент хочет вернуть товар без чека",
    "difficulty": "medium",
    "psychotype": "suspicious",
    "triggers": ["отказ в возврате", "просьба менеджера"],
    "initial_message": "Здравствуйте! Я покупал у вас телефон, но он сломался через неделю..."
  }'
```

**Генерация через AI Curator:**
```bash
curl -X POST http://localhost:8000/api/v1/curator/generate-scenario \
  -H "Authorization: Bearer <token>" \
  -d '{"topic": "отмена подписки", "difficulty": "hard"}'
```

Параметры сценария:

| Параметр | Описание | Пример |
|----------|----------|--------|
| title | Название сценария | «Возврат товара» |
| difficulty | Сложность: easy / medium / hard | medium |
| psychotype | Психотип клиента | suspicious / aggressive / confused / demanding |
| triggers | Список триггеров | ["отказ", "просьба менеджера"] |
| initial_message | Первое сообщение AI-клиента | «Здравствуйте! У меня проблема...» |

**Редактирование и удаление:**
- `PUT /api/v1/scenarios/{id}` — обновить параметры
- `DELETE /api/v1/scenarios/{id}` — удалить (только свои)

### 4.2 Группы операторов

**Создание группы:**
```bash
curl -X POST http://localhost:8000/api/v1/trainer/groups \
  -H "Authorization: Bearer <token>" \
  -d '{"name": "Ночная смена", "user_ids": ["uuid-1", "uuid-2", "uuid-3"]}'
```

**Просмотр результатов группы:**
```bash
curl http://localhost:8000/api/v1/analyst/stats?group_id=<group-uuid> \
  -H "Authorization: Bearer <token>"
```

**Что видит тренер:**
- Список операторов группы
- Средний score по каждому измерению
- Количество завершённых сессий
- XP и уровень каждого оператора
- Прогресс по учебным планам

### 4.3 Учебные планы и quiz

**Создание учебного плана:**
```bash
curl -X POST http://localhost:8000/api/v1/training-plans \
  -H "Authorization: Bearer <token>" \
  -d '{
    "title": "Работа с агрессивными клиентами",
    "scenario_ids": ["uuid-1", "uuid-2", "uuid-3"],
    "user_ids": ["uuid-10", "uuid-11"]
  }'
```

**Генерация quiz:**
```bash
curl -X POST http://localhost:8000/api/v1/curator/quiz \
  -H "Authorization: Bearer <token>" \
  -d '{"scenario_id": "<uuid>"}'
```

Quiz содержит 5 вопросов с вариантами ответов по заданному сценарию.

### 4.4 Аналитика для тренера

**Доступные отчёты:**

| Отчёт | Эндпоинт | Что показывает |
|-------|----------|----------------|
| Статистика группы | `GET /analyst/stats` | Средние score, XP, уровень по группе |
| Статистика оператора | `GET /analyst/stats?user_id=<uuid>` | Детальный профиль оператора |
| Прогресс плана | `GET /training-plans/{id}/progress` | Сколько сценариев пройдено |
| История сессий | `GET /sessions?user_id=<uuid>` | Все сессии оператора |

### 5.2 Fairness-отчёт

**Зачем:** Проверить, нет ли систематического перекоса в оценках
по полу, возрасту, акценту или родному языку.

**Запрос:**
```bash
curl http://localhost:8000/api/v1/analyst/fairness/report \
  -H "Authorization: Bearer <token>"
```

**4 метрики fairness:**

| Метрика | Что измеряет | Допустимое значение |
|---------|-------------|---------------------|
| Demographic Parity | Средний score одинаков для всех групп | Разница < 0.1 |
| Equalized Odds | Ошибки предсказания одинаковы для всех групп | Разница < 0.1 |
| Calibration | Score калиброван одинаково для всех групп | Разница < 0.1 |
| Disparate Impact | Соотношение positive rate между группами | > 0.8 |

**Что делать при алерте:**
1. Открой страницу `/admin/fairness` — увидишь детали
2. Определи, по какой группе перекос
3. Проверь сценарии: нет ли в них неявных предубеждений
4. При необходимости — отредактируй сценарии или настрой веса оценок

## 6. Типовые сценарии использования

### UC-1: Первая симуляция для новичка

**Роль:** Оператор
**Сценарий:** Confused client

1. Зайди на `/login`, введи логин и пароль
2. Нажми **New Session** → выбери **Confused client**
3. AI-клиент: «Здравствуйте! Я недавно подключил ваш интернет, но он работает очень медленно...»
4. Твой ответ: прояви эмпатию, уточни детали, предложи решение
5. Продолжай диалог (3-5 сообщений)
6. Нажми **Finish**
7. Посмотри оценку: какие dimensions сильные, какие слабые
8. Зайди в `/profile` — проверь XP и бейджи

### UC-2: Тренер создаёт сценарий для группы

**Роль:** Тренер
**Сценарий:** Работа с претензией

1. На `/trainer/scenarios` нажми **Create Scenario**
2. Заполни: title = «Претензия по качеству», difficulty = hard, psychotype = aggressive
3. Добавь триггер: «отказ в замене»
4. Напиши initial_message для AI-клиента
5. Сохрани сценарий
6. Создай группу «Ночная смена» на `/trainer/groups`
7. Добавь 3 операторов в группу
8. Назначь сценарий группе

### UC-3: Оператор получает бейдж

**Роль:** Оператор
1. Оператор проходит сессии в течение 5 дней подряд
2. На 5-й день система начисляет **Streak Master** бейдж
3. Оператор видит бейдж в `/profile`
4. XP: +50 за получение бейджа
5. Оператор поднимается на 1 место в leaderboard

### UC-4: Администратор проверяет fairness

**Роль:** Администратор
1. Зайди на `/admin/fairness`
2. Система показывает Demographic Parity = 0.15 (выше порога 0.1)
3. Детализация:的女性 (female) операторы имеют средний score 72, мужчины — 78
4. Проверь сценарии: нет ли в агрессивных сценариях гендерного перекоса
5. Отредактируй сценарии: убери гендерно-окрашенные триггеры
6. Запусти повторный fairness-отчёт — parity = 0.08 (OK)

### UC-5: Оператор с низким рейтингом получает учебный план

**Роль:** Тренер + Оператор
1. Тренер видит в `/analyst/stats` оператора со средним score 45
2. Тренер создаёт учебный план «Базовая эмпатия» с 3 лёгкими сценариями
3. Оператор видит план в `/sessions/plan`
4. Оператор проходит все 3 сценария за неделю
5. Тренер проверяет прогресс: score вырос с 45 до 68
6. Тренер добавляет ещё 2 средних сценария в план

### UC-6: Несколько операторов в одной сессии

**Роль:** Операторы A и B
(Планируется в Phase 5 — Voice + Multi-user)

1. Оператор A создаёт сессию и приглашает оператора B по ссылке
2. Оба оператора видят сообщения AI-клиента в реальном времени
3. Каждый может отправить сообщение (роль распределяется автоматически)
4. После завершения — каждый получает индивидуальную оценку

## 7. Часто задаваемые вопросы (FAQ)

### Регистрация и вход

**Q: Почему я не могу зарегистрироваться с паролем «12345»?**
A: Пароль должен содержать минимум 8 символов, хотя бы одну заглавную букву и хотя бы одну цифру.

**Q: Что делать, если забыл пароль?**
A: Функция сброса пароля планируется. Пока — обратись к администратору.

**Q: Почему я получил 401 через 30 минут?**
A: Access token живёт 30 минут. Используй refresh token для получения нового: `POST /auth/refresh`.

### Сессии

**Q: Почему после завершения сессии нет оценки?**
A: Если сессия прервалась (закрытие вкладки, таймаут), оценка не создаётся автоматически.
Нажми **Finish** перед выходом. Если оценка всё равно не появилась — вызови `POST /sessions/{id}/evaluate` вручную.

**Q: Сколько может длиться сессия?**
A: Максимум 60 минут. После этого сессия завершается принудительно.

**Q: Можно ли прервать сессию и продолжить позже?**
A: Нет, сессия — это непрерывный диалог. После завершения можно начать новую.

**Q: Почему AI-клиент стал агрессивнее?**
A: Это DDA (Dynamic Difficulty Adaptation). После успешных сессий сложность растёт.
Хорошая новость: после неудачных сессий сложность снижается.

### Оценка

**Q: Почему мой score по empathy низкий, хотя я был вежлив?**
A: Эмпатия — это не просто вежливость. Нужно валидировать эмоции клиента:
«Я понимаю, почему вы расстроены...», а не просто «Спасибо за обращение».

**Q: Как поднять score по structure?**
A: Используй нумерованные списки и чёткую структуру:
1) Понимаю вашу проблему...
2) Давайте проверим...
3) Если не поможет, я...

**Q: Почему оценка не появилась сразу?**
A: Coach-агент анализирует диалог асинхронно. Обычно это занимает 2-10 секунд.
Если прошло больше 30 секунд — проверь `/sessions/{id}` для статуса.

### XP и бейджи

**Q: Как быстро поднять уровень?**
A: Проходи сессии ежедневно (streak +200 XP), старайся набирать score >= 90 (+50 XP bonus).

**Q: Почему я не получил бейдж?**
A: Проверь условия в разделе 3.4. Некоторые бейджи требуют накопления (5 сессий, 10 сессий).

**Q: XP сбрасываются?**
A: Нет, XP накапливаются. Уровень только растёт (но могут быть корректировки при перебалансировке).

### Ошибки

**Q: Что делать при ошибке 429?**
A: Подожди 60 секунд. Это rate limit: 100 запросов в минуту на IP, 10 запросов в минуту на LLM.

**Q: Что делать при ошибке 500?**
A: Подожди 30 секунд и повтори. Если ошибка повторяется — сообщи администратору.

**Q: Данные сохраняются после перезапуска сервера?**
A: По умолчанию все данные in-memory — при перезапуске они теряются.
Если администратор подключил PostgreSQL и Redis — данные сохраняются.

## 8. Решение проблем

### 8.1 Проблемы с входом

| Проблема | Причина | Решение |
|----------|---------|---------|
| 401 Unauthorized | Token истёк | `POST /auth/refresh` |
| 401 после refresh | Refresh token тоже истёк | Зайди заново: `POST /auth/login` |
| 403 Forbidden | Недостаточно прав | Проверь свою роль (operator/trainer/admin) |
| Не приходит ответ на регистрацию | Email уже занят | Используй другой email |
| «Weak password» | Пароль не проходит валидацию | См. требования в разделе 2.1 |

### 8.2 Проблемы с сессиями

| Проблема | Причина | Решение |
|----------|---------|---------|
| Не загружается список сценариев | Нет сценариев в БД | Попроси тренера создать сценарий |
| AI-клиент не отвечает | LLM timeout | Подожди 10 секунд, обнови страницу |
| Сессия зависла | Ошибка соединения | Начни новую сессию |
| Оценка не появляется | Coach-агент не запустился | `POST /sessions/{id}/evaluate` вручную |
| Ошибка «Session not found» | Неверный ID сессии | Проверь URL или список сессий |

### 8.3 Проблемы с интерфейсом

| Проблема | Причина | Решение |
|----------|---------|---------|
| Страница не загружается | Фронтенд не запущен | Проверь `docker ps`, фронтенд на :5173 |
| Пустая страница после входа | Ошибка фронтенда | Открой консоль браузера (F12), проверь ошибки |
| Не обновляется leaderboard | Кэширование | Обнови страницу (F5) |
| Не отображаются бейджи | Бейдж ещё не получен | Проверь условия в разделе 3.4 |

### 8.4 Ошибки API (для разработчиков)

```bash
# Проверить health системы
curl http://localhost:8000/health

# Проверить свой token
curl http://localhost:8000/api/v1/auth/me \
  -H "Authorization: Bearer <token>"

# Проверить метрики
curl http://localhost:8000/api/v1/metrics
```
